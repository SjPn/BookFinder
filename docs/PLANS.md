# Планы развития Bookfinder

## ДНК книги — собственный индекс вместо жанров

**Статус:** запланировано  
**Приоритет:** высокий — главная дифференциация проекта  
**LLM:** локальная **Ollama** (без покупки API-токенов)

### Идея

Не использовать жанры как основной критерий поиска и рекомендаций. Для каждой книги автоматически строить **профиль «ДНК»** (JSON), который используется для:

- поиска и фильтрации по ощущениям, а не по полкам;
- рекомендаций «похожа по атмосфере / идеям / эмоциям»;
- накопления собственного индекса, независимого от рейтингов чужих библиотек.

Жанры остаются вспомогательным сигналом (~5% в формуле похожести), не ядром.

### Источники данных

| Источник | Что извлекаем |
|----------|----------------|
| **Текст книги** (FB2, где есть) | темп, диалоги, сложность языка; сюжетные оси: развитие героя, исследование мира, политика, экшен, жестокость, выживание, строительство, магия, наука |
| **Описание** | темы, идеи, тип героя, тип конфликта, сеттинг |
| **Отзывы** | что чаще отмечают, эмоции после прочтения, сильные/слабые стороны |
| **Эмбеддинги** (Ollama) | вектор из описания + отзывов + тегов → cosine similarity для смысловой похожести |

### Оси ДНК (шкала 1–10)

Все оси — плоский словарь `axes` в JSON. В UI — полоски/ползунки (как ████████).

#### Сюжет и содержание

| Ключ | Русское название | Заметка |
|------|------------------|---------|
| `character_growth` | Развитие героя | арка, взросление, смена убеждений |
| `world_exploration` | Исследование мира | открытие локаций, карт, тайн мира |
| `politics` | Политика | интриги, власть, фракции |
| `romance` | Романтика | любовная линия |
| `humor` | Юмор | комедия, ирония |
| `action` | Экшен | погони, драки, напряжённые сцены |
| `brutality` | Жестокость | насилие, мрачные сцены (см. также `darkness`) |
| `science` | Наука | научный подход, технологии, hard SF |
| `magic` | Магия | магическая система, заклинания |
| `survival` | Выживание | ресурсы, угрозы среды, «выжить любой ценой» |
| `construction` | Строительство | база, крафт, развитие поселения/империи (litRPG, progress fantasy) |

#### Атмосфера и опыт чтения

| Ключ | Русское название | Заметка |
|------|------------------|---------|
| `thinking` | Заставляет думать | идеи, головоломки |
| `philosophy` | Философия | экзистенциальные вопросы |
| `darkness` | Мрачность | тон, безнадёжность (не только насилие) |
| `hope` | Надежда | светлый тон, оптимизм |
| `psychology` | Психология | внутренний мир, мотивации |
| `worldbuilding` | Проработка мира | глубина лора (≠ исследование: мир может быть глубоким, но герой его не «открывает») |
| `realism` | Реализм | правдоподобие, быт |
| `pace` | Темп | динамика повествования |
| `difficulty` | Сложность языка | лексика, синтаксис |
| `dialogues` | Диалоги | доля и важность разговоров |
| `plot_twists` | Неожиданные повороты | twist'ы, непредсказуемость |

`brutality` и `darkness` разделены намеренно: книга может быть мрачной без жестокости и наоборот.

### Схема паспорта книги

```json
{
  "work_id": "fw:12345",
  "version": 1,
  "axes": {
    "character_growth": 8,
    "world_exploration": 7,
    "politics": 2,
    "romance": 1,
    "humor": 3,
    "action": 5,
    "brutality": 4,
    "science": 3,
    "magic": 3,
    "survival": 7,
    "construction": 6,
    "thinking": 10,
    "philosophy": 9,
    "darkness": 7,
    "hope": 4,
    "psychology": 8,
    "worldbuilding": 9,
    "realism": 3,
    "pace": 7,
    "difficulty": 6,
    "dialogues": 5,
    "plot_twists": 8
  },
  "labels": {
    "hero": "intelligent loner",
    "ending": "open",
    "conflict": "man vs system",
    "setting": "near-future Earth"
  },
  "themes": ["AI", "virtual reality", "identity", "future"],
  "reviews_summary": {
    "praised": ["идеи", "атмосфера"],
    "criticized": ["медленное начало"],
    "emotions": ["задумчивость", "тревога"]
  },
  "sources": {
    "annotation": 0.9,
    "reviews": 0.6,
    "text": 0.0
  },
  "embedding_model": "nomic-embed-text",
  "updated_at": "2026-07-06T00:00:00Z"
}
```

- `axes` — шкалы 1–10 (числовой профиль для фильтров и взвешенной похожести).
- `labels` — категориальные поля (герой, финал, конфликт).
- `sources` — уверенность по каждому источнику (для UI: «мало данных»).
- `version` — версия схемы; при смене промпта/модели — пересборка батчем.

**Хранение:** `data/processed/dna/{work_id}.json` + агрегат `data/processed/dna_index.json` (оси + путь к эмбеддингу для быстрых фильтров).

### Похожесть книг

Не один коэффициент, а взвешенная сумма:

| Доля | Компонент |
|------|-----------|
| 40% | эмбеддинг (cosine similarity) |
| 20% | темы (Jaccard / overlap) |
| 15% | эмоциональный профиль (расстояние по `axes`) |
| 10% | отзывы (summary + эмбеддинг отзывов) |
| 10% | стиль автора (профиль по другим книгам автора) |
| 5% | жанр (вспомогательно) |

Веса — стартовые; позже калибровать по кликам «похожие» и оценкам на портале.

### UI: фильтры вместо жанров

Вместо «Фантастика / Боевик» — оси опыта. На карточке книги — **полоски ДНК** (визуал как ████████).

**Сюжет и содержание:**

- 📈 Развитие героя (`character_growth`)
- 🗺️ Исследование мира (`world_exploration`)
- 🏛️ Политика (`politics`)
- ❤️ Романтика (`romance`)
- 😂 Юмор (`humor`)
- ⚔️ Экшен (`action`)
- 🩸 Жестокость (`brutality`)
- 🔬 Наука (`science`)
- ✨ Магия (`magic`)
- 🏕️ Выживание (`survival`)
- 🏗️ Строительство (`construction`)

**Атмосфера и стиль:**

- 🧠 Заставляет думать (`thinking`)
- 🌑 Мрачность (`darkness`)
- ⚡ Темп (`pace`)
- 🤯 Неожиданные повороты (`plot_twists`)
- 🌍 Проработка мира (`worldbuilding`)
- 👤 Сильный главный герой (label `hero`)
- 📚 Сложность языка (`difficulty`)

В сайдбаре — топ-N осей с быстрыми фильтрами; на странице книги — полный радар/полоски.

На странице книги — блоки рекомендаций:

- похожа по атмосфере;
- похожа по идеям;
- похожа по эмоциям;
- похожа по стилю;
- похожа по динамике;
- читателям этой книги также понравились.

Текущий `similar_works()` (жанр + автор + fuzzy title) заменяется/дополняется DNA-слоем.

## Принцип реализации

### Архитектура

```
[аннотация / отзывы / FB2]
        ↓
  scripts/build_dna.py  ←── Ollama (локально)
        ↓
  data/processed/dna/*.json
  data/processed/dna_embeddings/  (или sqlite-vec)
        ↓
  export_runtime_catalog.py  →  dna_summary в runtime (оси для фильтров)
        ↓
  catalog.similar_works_dna()  +  API /api/works/{id}/similar?mode=atmosphere|ideas|...
        ↓
  web: ползунки-фильтры + блоки «похожие»
```

**Важно для Render (512 MB RAM):** полный каталог эмбеддингов в память не грузить. Поиск top-K — через файл на диске (`sqlite-vec`, `hnswlib`) или предрасчёт top-N соседей в `dna_neighbors.json` при офлайн-сборке.

### Ollama (локально, без платных токенов)

| Задача | Модель (пример) | Примечание |
|--------|-----------------|------------|
| Извлечение JSON-профиля | `qwen2.5:7b` или `llama3.1:8b` | structured output, промпт с жёсткой схемой |
| Эмбеддинги | `nomic-embed-text` или `mxbai-embed-large` | `POST http://localhost:11434/api/embeddings` |
| Суммаризация отзывов | та же chat-модель | батч по work_id, кэш в `dna/*.json` |

**Клиент:** `src/bookfinder/ollama_client.py` — обёртка с retry, таймаутом, `OLLAMA_HOST` из env (по умолчанию `http://127.0.0.1:11434`).

**Сборка только локально / на VPS (Hetzner):** Render не запускает Ollama. Пайплайн `build_dna.py` гоняется на машине с GPU/CPU, артефакты (`dna/`, эмбеддинги) коммитятся в `data/processed/` или заливаются отдельно — как сейчас с каталогом.

**Батчинг:** очередь work_id, пропуск если `dna/{id}.json` уже есть и `version` совпадает; `--force` для пересборки.

**Текст FB2:** не весь роман в промпт — выборка (начало + середина + конец, ~8–12k символов) + эвристики (диалоги по «—», сложность по индексу читабельности) без LLM где возможно.

### Фазы

#### Фаза 0 — инфраструктура

- [ ] `src/bookfinder/ollama_client.py`
- [ ] JSON Schema / Pydantic-модель `BookDNA`
- [ ] `scripts/build_dna.py --limit 500 --sources annotation`
- [ ] Пилот на 500–1000 книг

#### Фаза 1 — MVP (аннотация + Ollama)

- [ ] Профиль `axes` + `themes` + `labels` из описания
- [ ] Эмбеддинги описаний через Ollama
- [ ] `similar_works_dna()` — cosine + themes
- [ ] API `GET /api/works/{id}/similar?mode=ideas`
- [ ] UI: один блок «Похожие по идеям» на странице книги

#### Фаза 2 — отзывы

- [ ] Агрегация `reviews_summary` из `data/processed/reviews/works.json`
- [ ] Эмбеддинг отзывов (суммаризация → embed)
- [ ] Вес 10% в формуле похожести

#### Фаза 3 — текст FB2

- [ ] Парсер выборки из `data/books/fb2/`
- [ ] Эвристики: pace, dialogues, difficulty
- [ ] LLM-дозаполнение осей по выборке текста
- [ ] Обновление `sources.text`

#### Фаза 4 — фильтры и полная похожесть

- [ ] Ползунки-фильтры в UI (оси из `dna_index.json`)
- [ ] Все режимы рекомендаций (атмосфера, эмоции, динамика, …)
- [ ] Стиль автора (агрегат DNA по книгам автора)
- [ ] sqlite-vec / предрасчёт соседей для продакшена
- [ ] Калибровка весов (опционально: логи кликов)

### Риски и митигация

| Риск | Решение |
|------|---------|
| Дрейф шкал между прогонами | `version` + фиксированный промпт; пересборка целиком при смене |
| Медленная Ollama на 40k книг | ночной батч, инкремент, приоритет популярным |
| Нет FB2 у части книг | профиль из аннотации + отзывов (`sources` отражает пробел) |
| Render OOM | DNA summary в runtime JSON; тяжёлые эмбеддинги — только на диске / top-K офлайн |

### Что уже есть в проекте

- ~40k книг, аннотации (`works_details.json`), отзывы (`reviews/works.json`)
- ~21k FB2 локально (растёт через `fetch_fb2.py`)
- `similar_works()` — жанр/автор (заменить DNA-слоем)
- Очищенные жанры (`genre_filter.py`) — остаются как 5% сигнала
